# <Spec Title> Модель данных

## Scope

- Связанные `AC-*`: `AC-001`
- Связанные `DEC-*`: `DEC-001`
- Статус: `changed` или `no-change`
- Явно: если значимого изменения data model не требуется

## Сущности

### DM-001 Сущность 1

- Назначение:
- Источник истины:
- Инварианты:
- Связанные `AC-*`:
- Связанные `DEC-*`:
- Поля:
  - `field_name` - тип/shape, required/optional, смысл, default/validation rule когда важно
- Жизненный цикл:
  - когда и каким действием создаётся
  - когда и каким действием обновляется
  - когда удаляется, истекает или архивируется
- Замечания по консистентности:
  - какое stale/duplicated/out-of-order/partially-written состояние недопустимо

### DM-002 Сущность 2 — та же структура (Назначение / Источник истины / Инварианты / Поля / Жизненный цикл / Замечания по консистентности)

## Связи

- `DM-001 -> DM-002`: связь, ownership или cardinality
- Явно: если значимых межсущностных связей нет

## Производные правила

- Вычисляемое значение, правило нормализации или правило перехода состояния
- Явно: если производные правила не нужны

## Переходы состояний

- Trigger/event -> предыдущее состояние -> следующее состояние
- Validation/guard, не дающий выполнить некорректный переход
- Явно: если жизненный цикл достаточно прост и отдельный список не нужен

## Вне scope

- Сущность, поле или состояние, намеренно не моделируемые здесь

## No-Change Stub

Используйте эту секцию вместо удаления файла, если фича не меняет модель:

- Статус: `no-change`
- Причина: фича не добавляет и не меняет persisted entities, value objects, state transitions или contract-relevant payload shapes
- Revisit triggers:
  - появляется новое сохраняемое состояние
  - появляются новые инварианты или lifecycle states
  - API/event payload shape нужно отслеживать именно здесь
